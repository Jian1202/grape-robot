# 01-项目总览与技术路线

本文件长期维护，不每天新建。  
用于记录“基于移动机械臂的葡萄智能视觉夹取系统”项目的总体目标、技术路线、阶段划分和当前工程决策。

---

## 1. 项目名称

```text
基于移动机械臂的葡萄智能视觉夹取系统
```

------

## 2. 项目背景

本项目围绕 ROSLander 移动机械臂平台展开，目标是在指定机器人硬件上完成葡萄目标识别、三维定位与机械臂夹取。

根据当前已知信息，项目涉及的主要硬件包括：

```
Astra Pro Plus 相机
Gemini 双目 / 深度相机
移动底盘
机械臂
夹爪
Jetson 嵌入式平台
ROSLander 机器人本体
```

其中：

```
Astra Pro Plus 相机：可能用于环境视觉或前方视觉
Gemini 相机：当前理解为机械臂末端相机，可能用于近距离识别与深度定位
移动底盘：用于机器人移动
机械臂与夹爪：用于抓取葡萄目标
Jetson 平台：用于运行视觉识别、ROS2 节点或系统集成程序
```

需要后续通过源码、实验和官方文档进一步确认。

------

## 3. 项目总目标

项目最终目标是实现一套可以演示的葡萄智能视觉夹取系统。

理想完整流程如下：

```
机器人启动
↓
相机获取图像
↓
视觉模型识别葡萄目标
↓
判断葡萄成熟状态
↓
获取目标中心点与深度信息
↓
计算葡萄三维坐标
↓
机械臂运动学求解
↓
机械臂移动到目标附近
↓
夹爪完成夹取
↓
系统完成一次葡萄夹取演示
```

第一阶段不追求完整农业导航任务，而是优先完成一个最小可演示闭环：

```
看见葡萄
↓
定位葡萄
↓
机械臂靠近
↓
夹爪动作
```

------

## 4. 当前项目理解

本项目不是从零编写一个完整采摘机器人系统，而是采用：

```
先复现
↓
再替换
↓
最后集成
```

的工程路线。

含义如下：

```
先复现：优先跑通官方已有 ROS2 抓取闭环或示例程序
再替换：把官方颜色识别替换为 YOLO 葡萄识别
最后集成：把识别、深度定位、机械臂控制和夹爪动作整合成演示系统
```

这样做的原因：

```
官方示例已经包含相机、深度、机械臂、运动学、夹爪等关键模块
直接从零写完整系统风险高
先复用官方闭环可以更快找到可运行路径
YOLO 识别应作为视觉模块替换进去，而不是一开始直接控制机械臂
```

## 6. 系统目标链路

当前设想的最终系统链路如下：

```
Gemini / Astra 图像
↓
YOLO 葡萄检测
↓
输出检测框
↓
选择目标葡萄
↓
取检测框中心点 u, v
↓
读取深度图 depth(u, v)
↓
结合相机内参计算三维坐标 X, Y, Z
↓
坐标转换到机械臂坐标系
↓
机械臂运动学求解
↓
发送机械臂控制命令
↓
夹爪闭合
↓
完成夹取
```

其中三维坐标计算的基础形式为：

```
Z = depth(u, v)
X = (u - cx) * Z / fx
Y = (v - cy) * Z / fy
```

说明：

```
u, v：图像中的目标中心点像素坐标
depth(u, v)：该点对应的深度值
fx, fy, cx, cy：相机内参
X, Y, Z：相机坐标系下的三维坐标
```

当前还没有确认：

```
相机坐标系到机械臂坐标系的完整转换方式：不知道
track_and_grab 中是否已经封装了坐标转换：不知道
YOLO 检测结果应该接入哪个具体函数：不知道
```

## 8. YOLO 在项目中的定位

YOLO 是项目中的视觉识别模块，不是机械臂控制模块。

当前 YOLO 的作用定位：

```
输入图像
↓
识别葡萄
↓
输出检测框和类别
↓
为深度定位提供目标中心点
```

YOLO 不直接负责：

```
机械臂运动学求解
夹爪开合
底盘导航
深度图读取
相机坐标转换
```

当前 YOLO 支线状态：

```
已跑通过 YOLO 官方示例
还没有整理葡萄数据集
还没有训练葡萄模型
还没有接入 ROS2 抓取闭环
```

第一版 YOLO 任务建议：

```
目标检测
```

第一版类别建议：

```
ripe_grape
unripe_grape
```

暂时不优先做：

```
实例分割
果粒计数
复杂成熟度评分
多阶段成熟度打分
```

原因：

```
第一阶段目标是先完成系统闭环
类别和任务越复杂，数据集整理和模型训练风险越高
先完成检测框输出，更容易接入现有抓取流程
```

## 9. 机械臂控制在项目中的定位

机械臂控制是最终执行模块。

当前已经确认：

```
可以通过 /servo_controller 控制机械臂舵机
id10 是夹爪舵机
id10=300 会产生明显夹爪动作
id10=500 是 init 默认值
```

当前理解的机械臂控制链路：

```
/servo_controller
↓
controller_manager / servo_manager
↓
/ros_robot_controller/bus_servo/set_position
↓
ros_robot_controller
↓
底层总线舵机
↓
机械臂 / 夹爪动作
```

项目中的机械臂控制不建议从底层直接写起。

推荐方式：

```
优先复用官方 /servo_controller
优先研究官方动作组和 track_and_grab 调用方式
暂时不要直接操作底层 bus_servo 接口
```

仍需确认：

```
机械臂主体 id1~id5 的精确物理对应关系：不知道
每个舵机完整安全 pulse 范围：不知道
官方抓取动作组和 track_and_grab 的关系：不知道
```

## 13. 当前关键决策

### 13.1 不从零写完整抓取程序

原因：

```
官方已有 ROS2 抓取闭环
从零写成本高、风险高、调试周期长
```

------

### 13.2 不马上把 YOLO 接机械臂

原因：

```
YOLO 模型还没有葡萄数据
机器人抓取闭环源码尚未读懂
不知道 YOLO 输出应该接到哪个模块
```

------

### 13.3 不优先移植 ROS1 农业案例

原因：

```
当前环境是 ROS2 Humble
ROS1 命令和架构不能直接照搬
```

------

### 13.4 优先复用 / 改造官方 ROS2 流程

```
复用官方 track_and_grab 的深度定位和机械臂控制部分
```
