# configs

本目录用于保存需要由多个模块复用、需要审查和可回滚的项目配置，避免把阈值、路径、速度和硬件参数散落在代码中。

## 计划中的配置

| 文件 | 用途 | 创建前必须确认 |
| --- | --- | --- |
| `camera_topics.yaml` | RGB、depth、camera_info 话题与图像约定 | 实机 topic 类型、分辨率、encoding |
| `yolo_grape.yaml` | 模型、类别、置信度和输入尺寸 | 模型哈希、类别表、现场验证结果 |
| `arm_params.yaml` | 工作空间、速度、舵机和安全姿态 | 坐标系、单位、实机安全范围 |
| `nav_points.yaml` | 教室导航点和接近距离 | 地图版本、坐标系、现场测量 |

这些文件目前是规划项，不代表已经存在或已经验证。不得为了让代码“先跑起来”而由 Codex自行填写硬件数值。

## 每个参数至少包含

```text
名称
含义
单位
默认值
最小/最大值
来源
验证日期
适用设备或版本
修改原因
```

## 管理规则

- 不保存密码、密钥、账号和私人网络信息。
- 不写死开发者电脑的绝对路径；机器人部署路径必须说明环境。
- 修改硬件参数必须单独提交，不能混入普通重构。
- 未验证参数使用明确占位和 `UNKNOWN`，不能给出看似合理的默认数值。
- 参数影响实车动作时，按[开发与实车放行关卡](../docs/governance/DEVELOPMENT_GATES.md)重新验证。
- 旧参数需要保留在 Git 历史或版本化配置中，确保能够回滚。

## Codex 使用要求

Codex 新增或修改配置前，必须先说明参数来源、单位、合法范围、使用位置和测试方法。缺少其中任意一项时停止实现。
