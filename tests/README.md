# tests

本目录用于保存项目自己的自动测试、固定小样本和测试说明。目标是尽可能在代码接触真实小车前发现问题。

## 建议结构

```text
tests/
├── unit/          纯算法、状态机、配置解析
├── contract/      ROS2 接口名称、消息类型和外部符号合同
├── fixtures/      少量固定 RGB/深度样本和元数据
├── replay/        rosbag 或录制数据回放说明与脚本
└── hardware/      实车 Gate 检查脚本和人工记录模板
```

目前这些子目录尚未全部建立，不代表已有测试覆盖。

## 优先补充的测试

1. 多目标选择和目标框边界；
2. 深度有效像素、中位数、离群值、空 ROI 和 NaN；
3. RGB 与深度尺寸不同时的拒绝或映射行为；
4. 连续帧稳定判断；
5. 状态机超时、取消、目标丢失和恢复；
6. IK 无解、服务无响应和返回 pulse 数量异常；
7. 所有执行器开关为 `false` 时不会发布动作命令；
8. 新增 ROS2 接口是否已登记在接口合同中。

## 数据管理

- 不提交大量测试图片、完整 rosbag 或大模型。
- 小样本必须记录来源、尺寸、深度单位、预期结果和是否可公开。
- 大文件保存在私发、网盘、Release 或 Git LFS 中，仓库保存哈希和获取说明。
- 测试不能依赖某位组员电脑上的绝对路径。

## 通过标准

语法检查通过不等于功能通过，离线测试通过也不等于实车安全。测试结果必须结合[开发与实车放行关卡](../docs/governance/DEVELOPMENT_GATES.md)解释。

Codex 完成测试任务时必须报告：运行命令、通过/失败数量、未运行原因、未覆盖风险，以及测试是否可能触发硬件动作。
