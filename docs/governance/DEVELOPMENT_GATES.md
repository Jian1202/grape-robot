# 开发与实车放行关卡

本文件规定代码从“想法”进入“真实小车动作”的最低条件。它不能消除所有风险，但可以阻止未经验证的代码直接控制执行器。

## 1. 总原则

```text
没有证据，不进入实现
没有测试，不进入 ROS2 联调
没有回放，不进入实车感知
没有低风险单项验证，不进入完整抓取
```

任何关卡失败后都应停止、保存日志并回退，不得通过提高速度、扩大位移或关闭检查强行通过。

## 2. Gate 0：任务与事实校准

通过条件：

- 已读 `AGENTS.md` 和相关治理文档；
- 已确认仓库、分支和工作区状态；
- 已列出修改文件白名单；
- 所有外部接口都有来源或明确标为 `UNKNOWN`；
- 已写验收条件和回滚方式。

禁止事项：修改代码后再补设计、凭旧案例猜 ROS2 接口。

## 3. Gate 1：静态与单元测试

适用：纯算法、状态机和配置解析。

最低检查：

- Python 语法编译通过；
- 目标模块在正确环境中可导入；
- 相关单元测试通过；
- 空输入、越界、NaN、无效深度、服务失败和超时有覆盖；
- 不连接 ROS2 或执行器也能验证核心逻辑。

产物：测试输出、用例说明、未覆盖项。

## 4. Gate 2：固定数据或 rosbag 回放

适用：检测、中心点、深度、目标稳定和状态转换。

通过条件：

- 使用固定 RGB/深度样本或 rosbag 可重复运行；
- 记录预期目标、中心点、有效深度比例和输出；
- 叶片遮挡、空景、小目标和无效深度有测试；
- 不发送任何硬件控制消息；
- 多次运行结果符合预设容差。

大型 rosbag 不提交普通 Git，仓库只保存来源、哈希、获取方式和小型必要样本。

## 5. Gate 3：实车检测-only

要求：

- `enable_arm=false`；
- 底盘、夹爪、脱离动作全部关闭；
- 现场人员确认周围安全；
- 只显示检测框、目标中心、深度和计划动作；
- 验证 RGB/深度分辨率、时间同步和坐标边界；
- 保存完整终端日志和代表性画面。

当前安全视觉测试可参考[组员小车安全视觉测试单](../reference/09-组员小车安全视觉测试单.md)。

## 6. Gate 4：机械臂跟踪-only

前置条件：Gate 3 通过，机械臂安全范围和停止方式已确认。

要求：

- 只允许低速、小范围跟踪；
- 禁止夹爪闭合、向下脱离和底盘移动；
- 目标丢失立即停止，不使用旧目标；
- IK 无解、深度异常或服务超时立即退出；
- 现场人员可随时停止程序和断开执行器；
- 验证相机移动方向与画面误差收敛方向一致。

## 7. Gate 5：夹爪单项测试

前置条件：夹爪周围无物体、线缆和人手，id10 语义已重新确认。

按顺序验证：

1. 当前状态读取；
2. 小幅张开；
3. 回到已验证安全位置；
4. 小幅闭合；
5. 保持；
6. 停止或异常后的归位。

记录每个 pulse 的物理状态、方向、耗时、反馈和安全范围。未经确认不能把 200/300/500/540 命名为固定动作。

## 8. Gate 6：低速脱离单项测试

前置条件：世界坐标方向、IK、下方空间、夹爪保持和停止方式均已验证。

要求：

- 仅使用假葡萄；
- 从低速、约 2cm 小位移开始；
- 每次只改变一个参数；
- 下拉前确认夹爪状态；
- 动作后检查目标是否脱离；
- 失败时停止并回安全姿态；
- 通过后再讨论 3至5cm，不直接照搬旧代码的 5cm。

## 9. Gate 7：底盘预对准

要求：

- 单独安全开关；
- 已确认真实底盘控制入口和坐标方向；
- 机械臂保持安全姿态；
- 低速短距离测试；
- 有最大移动时间、距离和停止条件；
- 目标丢失时不继续盲走。

## 10. Gate 8：完整采摘集成

只有前述关卡分别通过后，才允许组合：

```text
SEARCH
-> BASE_ALIGN
-> ARM_TRACK
-> LOCALIZE
-> CHECK_IK
-> PREGRASP
-> VISUAL_REFINE
-> CLOSE_GRIPPER
-> PULL_DOWN
-> VERIFY_DETACH
-> PLACE
-> RESET
```

每个状态必须包含：

- 输入和输出；
- 进入和退出条件；
- 超时；
- 可观察日志；
- 失败原因；
- 转入 `RECOVER` 的路径。

## 11. 立即停止条件

出现以下任一情况立即停止当前关卡：

- 未知接口或消息字段；
- RGB/深度尺寸不一致且没有映射；
- 深度大面积无效或跳变；
- IK 无解或返回数量异常；
- 服务无响应；
- 执行器方向与预期相反；
- 舵机反馈不跟随命令；
- 目标丢失后仍继续动作；
- 夹爪、机械臂、相机、线缆或底盘接近碰撞；
- 现场人员无法立即停止；
- 实际代码与本次审查提交不一致。
