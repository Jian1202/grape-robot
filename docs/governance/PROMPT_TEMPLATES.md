# Codex 提示词模板

这些模板用于统一组内使用方式。使用时替换方括号内容，不要删掉事实来源、文件白名单和安全限制。

## 1. 新任务只读校准

```text
请先完整阅读仓库根目录 AGENTS.md，并按其中的必读顺序阅读相关项目文档。

本轮先只读分析，不修改文件，不提交，不推送，也不运行任何会驱动底盘、机械臂或夹爪的命令。

任务：[填写要分析的问题]

请先输出：
1. 仓库根目录、当前分支和工作区状态；
2. 与任务有关的真实文件和调用链；
3. 每个外部 import、函数、ROS2 topic/service/message 的定义来源；
4. VERIFIED_REPO、DOCUMENTED_ROBOT、REFERENCE_ONLY、INFERRED、UNKNOWN 对照表；
5. 当前证据能确定的原因和不能确定的偶然因素；
6. 如果需要实机信息，只给出只读验证命令；
7. 最小、可验证的下一步。

禁止根据名称猜测接口或硬件行为。找不到定义时必须标为 UNKNOWN。
```

## 2. 写代码前的设计评审

```text
继续遵守 AGENTS.md。本轮只做设计，不修改文件。

目标：[填写单一目标，例如“实现稳健深度采样函数”]

请提供：
1. 准备修改的准确文件白名单；
2. 保持不动的文件和行为；
3. 新增或修改的函数/类签名；
4. 每个函数的调用方、输入、输出、单位、异常和超时；
5. 所有外部符号的证据来源；
6. 单元测试和固定数据用例；
7. 对 ROS2 和实车的影响；
8. 回滚方式；
9. 仍为 UNKNOWN 的内容。

不要扩展为整套系统重写。
```

## 3. 限定范围实现

```text
请实现已经评审通过的目标：[填写目标]。

只允许修改：
- [文件 1]
- [文件 2]

禁止修改：
- official_examples/
- docs/source-materials/
- [其他不在本次范围内的文件]

安全要求：
- 不新增未经登记的 ROS2 接口；
- 不新增无定义来源的外部 import 或函数；
- 不改变任何舵机 pulse、机械臂位移、速度或坐标偏移；
- 所有执行器开关保持默认 false；
- 不运行真实硬件动作。

验收：
- [测试 1]
- [测试 2]
- [边界条件]

修改完成后报告实际文件、接口来源、测试结果、未验证项和回滚方式。
如果发现现有设计依赖 UNKNOWN，请停止实现并说明缺少什么。
```

## 4. 独立代码审查

```text
请先读 AGENTS.md、项目事实基线和 ROS2 接口合同。
将当前改动视为可能导致机械臂、夹爪或底盘误动作的代码进行独立审查，不要默认实现者的假设正确。

重点检查：
1. 是否存在没有定义来源的 import、函数、类、topic、service、message 或字段；
2. 是否把 ROS1、厂商旧案例或文档建议写成当前 ROS2 已实现；
3. 是否混淆米、毫米、像素、角度、弧度或 pulse；
4. 是否混淆 RGB 与深度分辨率、坐标系或时间戳；
5. 是否处理空目标、越界、无效深度、NaN、IK 无解和服务超时；
6. 目标丢失后是否仍使用旧目标；
7. 关闭安全开关后是否仍可能发布硬件命令；
8. 是否具备停止、异常复位和回滚路径；
9. 测试是否真的覆盖行为，而不是只检查语法；
10. 哪些结论当前无法证明。

按严重程度列问题，给出文件和位置。没有证据时不要替实现者补接口。
```

## 5. 实机接口取证

```text
请先读 AGENTS.md 和 ROS2_INTERFACE_CONTRACT.md。
目标是补齐 [接口名称] 的真实证据，本轮禁止发送任何运动命令。

请只给出：
1. 需要在机器人上执行的只读命令；
2. 每条命令用于确认什么；
3. 输出应保存到 docs/evidence/ 下的哪个文件；
4. 哪些敏感信息需要脱敏；
5. 拿到输出后如何更新事实状态。

不要提供 ros2 topic pub、舵机动作、底盘速度或机械臂移动命令。
```

## 6. 文档与资料整理

```text
请先读 AGENTS.md 和 docs/README.md。本轮只整理资料，不修改机器人运行代码。

资料范围：[填写目录或文件]
允许修改：[填写 Markdown 白名单]

要求：
1. 所有结论带来源路径或源码位置；
2. 区分 ROS1、ROS2、厂商资料和当前自研代码；
3. 区分可直接复用、需要重写、仅供参考和未知；
4. 不复制大段源码，整理状态、输入、输出、阈值和失败条件；
5. 更新导航 README；
6. 不把建议写成已实现。
```

## 7. Gate 实车验证前检查

```text
请根据 DEVELOPMENT_GATES.md 检查本次是否具备进入 Gate [编号] 的条件。

输入材料：
- Git commit：[填写]
- 配置：[填写]
- 模型及哈希：[填写]
- 已通过关卡：[填写]
- 日志/回放：[填写]

只做放行检查，不运行硬件动作。输出：
1. 已满足条件；
2. 缺失证据；
3. 必须停止的问题；
4. 现场检查清单；
5. 结论：可以进入 / 不可以进入。

证据不足时必须判定“不可以进入”。
```

## 8. 发现模型开始自由发挥时

```text
停止当前实现。请指出你刚才使用的每个外部符号和硬件参数的真实定义来源。
找不到来源的全部改为 UNKNOWN，并撤回基于它们的代码方案。
回到只读阶段，说明需要补充的源码、接口输出或实机证据。
```
