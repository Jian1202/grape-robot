# 组员小车安全视觉测试单

本次只测试 YOLO 葡萄识别画面。机械臂不跟踪、不抓取，底盘不移动。

## 1. 本次使用的模型

模型文件由组内单独传递，不在 GitHub 仓库中：

```text
grape_v3_sceneholdout_cpu_e20/weights/best.pt
文件大小：5,424,218 bytes
SHA-256：d3c3173f1c4001461baeb2829bd543271aa3dcf5c22c9dd4a5e377163348beee
```

类别：

```text
0: unripe_grape  绿色葡萄
1: ripe_grape    紫红葡萄
```

## 2. 上传并切换 v3 模型

在本地终端把收到的 `best.pt` 上传到机器人临时目录。将 `/本地路径/` 替换为模型实际位置：

```bash
scp /本地路径/best.pt \
  ubuntu@ubuntu.local:/tmp/grape_v3_sceneholdout.pt
```

在机器人终端执行：

```bash
PROJECT=~/teams/ctrlteam/grape_robot
MODEL_NAME=grape_v3_sceneholdout_20260711.pt

mkdir -p "$PROJECT/models/archive"
cp /tmp/grape_v3_sceneholdout.pt \
  "$PROJECT/models/archive/$MODEL_NAME"

sha256sum "$PROJECT/models/archive/$MODEL_NAME"
```

哈希必须是：

```text
d3c3173f1c4001461baeb2829bd543271aa3dcf5c22c9dd4a5e377163348beee
```

确认模型能加载：

```bash
python3 - <<'PY'
from ultralytics import YOLO

path = "/home/ubuntu/teams/ctrlteam/grape_robot/models/archive/grape_v3_sceneholdout_20260711.pt"
model = YOLO(path)
print("MODEL_OK")
print(model.names)
PY
```

确认输出包含 `unripe_grape` 和 `ripe_grape` 后，再切换当前模型：

```bash
ln -sfn "archive/$MODEL_NAME" "$PROJECT/models/current.pt"
readlink -f "$PROJECT/models/current.pt"
```

旧模型仍保留在 `models/archive/`，可随时回滚。

## 3. 安全视觉模式启动

必须在 NoMachine 图形桌面的终端执行。首次使用 416 输入尺寸：

```bash
IMGSZ=416 CONFIDENCE=0.4 TARGET_CLASS=ripe_grape \
  ~/teams/ctrlteam/grape_robot/scripts/run_vision.sh
```

识别绿色葡萄：

```bash
IMGSZ=416 CONFIDENCE=0.4 TARGET_CLASS=unripe_grape \
  ~/teams/ctrlteam/grape_robot/scripts/run_vision.sh
```

如果 416 画面明显卡顿，再切换到 320：

```bash
IMGSZ=320 CONFIDENCE=0.4 TARGET_CLASS=ripe_grape \
  ~/teams/ctrlteam/grape_robot/scripts/run_vision.sh
```

运行脚本默认传入：

```text
enable_arm=false
```

因此本次只显示检测框、标签、置信度和中心点。不要手动改成 `enable_arm=true`。

## 4. 现场测试顺序

每次只选择一个目标类别。按以下顺序记录画面：

1. 清晰近距离葡萄。
2. 叶片遮挡的葡萄。
3. 远距离、小目标葡萄。
4. 空画面、桌面、椅子、线缆和红色物体。
5. 绿色与紫红葡萄混合场景，分别切换两个 `TARGET_CLASS` 测试。

每个场景保存截图或短视频，并记录是否出现：

```text
漏检
误检
框偏移
画面卡顿
终端报错
```

已知重点：紫红葡萄的重遮挡、极小目标，以及空景中的红色物体最值得观察。

## 5. 退出与回滚

退出优先级：

```text
图像窗口按 q 或 Esc
终端按 Ctrl+C
```

`run_vision.sh` 退出后会尝试恢复默认 App 服务。

如需回滚，先查看历史模型：

```bash
ls -lh ~/teams/ctrlteam/grape_robot/models/archive/
```

然后切换到旧模型：

```bash
cd ~/teams/ctrlteam/grape_robot/models
ln -sfn "archive/旧模型文件名.pt" current.pt
```

## 禁止事项

- 不要传入 `enable_arm=true`。
- 不要直接修改 `ros2_ws` 里的运行副本；`run_vision.sh` 会从项目主目录同步代码。
- 不要删除 `models/archive/` 中的旧模型。
- 遇到模型加载、相机或 ROS2 报错时，保留终端完整输出再反馈。
