## 4. `.d6a` 文件是什么

当前已确认：

```text
.d6a 文件本质上是 SQLite 数据库文件
```

它不是普通文本文件。

如果用普通文本编辑器打开，可能出现：

```text
乱码
空白
不可读内容
```

正确查看方式：

```text
使用 sqlite3 工具
或使用 Python 标准库 sqlite3 读取
```

当前机器人上曾出现：

```text
zsh: command not found: sqlite3
```

说明：

```text
系统没有安装 sqlite3 命令行工具
但 Python sqlite3 标准库可用
```

已验证：

```bash
python3 - <<'PY'
import sqlite3
print("python sqlite3 ok")
PY
```

输出：

```text
python sqlite3 ok
```

## 5. 动作组读取源码

动作组读取源码路径：

```text
/home/ubuntu/ros2_ws/src/driver/servo_controller/servo_controller/servo_controller/action_group_controller.py
```

当前理解：

```text
action_group_controller.py 负责读取 .d6a 动作组文件
然后把动作组中的每一帧转换为 /servo_controller 消息发布出去
```

核心流程：

```text
打开 .d6a 文件
↓
读取 ActionGroup 表
↓
逐行读取 Time、Servo1~Servo6
↓
生成 ServosPosition 消息
↓
position_unit = pulse
↓
duration = Time / 1000
↓
发布到 /servo_controller
↓
等待当前帧动作执行完成
↓
继续下一帧
```

重要结论：

```text
动作组最终还是通过 /servo_controller 控制机械臂
```

这说明：

```text
手动发布 /servo_controller 命令
和执行动作组
本质上都进入同一条机械臂控制链路
```

## 6. 动作组与控制链路关系

当前确认的动作组控制链路：

```text
.d6a 动作组文件
↓
action_group_controller.py 读取
↓
发布 /servo_controller
↓
controller_manager
↓
servo_manager
↓
/ros_robot_controller/bus_servo/set_position
↓
ros_robot_controller
↓
底层总线舵机
↓
机械臂 / 夹爪动作
```

其中：

```text
.d6a 文件：保存预设动作
action_group_controller.py：读取并执行动作
/servo_controller：上层机械臂控制入口
底层 bus_servo：真正驱动舵机
```


字段含义当前理解：

```text

Servo1：第 1 个舵机目标位置

Servo4：第 4 个舵机目标位置
Servo5：第 5 个舵机目标位置
Servo6：第 6 个动作组舵机值，源码中会映射为 id10
```

是否所有 `.d6a` 文件表结构都完全相同：

```text
暂未全部验证
```

但目前读取过的动作组基本符合该结构。

## 11. `init.d6a` 对应的安全保持命令

可以用以下命令向 `/servo_controller` 发布 init 姿态：

```bash
ros2 topic pub --once /servo_controller servo_controller_msgs/msg/ServosPosition "{
  duration: 1.0,
  position_unit: 'pulse',
  position: [
    {id: 1, position: 500.0},
    {id: 2, position: 765.0},
    {id: 3, position: 15.0},
    {id: 4, position: 220.0},
    {id: 5, position: 500.0},
    {id: 10, position: 500.0}
  ]
}"
```

用途：

```text
验证 /servo_controller 链路
让机械臂回到 init 已知姿态
作为后续测试前后的安全参考姿态
```

注意：

```text
是否任何情况下都安全回 init：不知道
如果机械臂当前姿态距离 init 很远，仍可能发生较大动作
执行前应确认周围安全
```

## 19. Python 读取动作组方法

### 19.1 读取单个动作组

示例：读取 `init.d6a`

```bash
python3 - <<'PY'
import sqlite3

path = "/home/ubuntu/software/arm_pc/config/ActionGroups/init.d6a"

conn = sqlite3.connect(path)
cur = conn.cursor()

print("=== tables ===")
cur.execute("SELECT name FROM sqlite_master WHERE type='table';")
print(cur.fetchall())

print("\n=== ActionGroup table info ===")
cur.execute("PRAGMA table_info(ActionGroup);")
for row in cur.fetchall():
    print(row)

print("\n=== first 5 rows ===")
cur.execute("SELECT * FROM ActionGroup LIMIT 5;")
for row in cur.fetchall():
    print(row)

cur.close()
conn.close()
PY
```

用途：

```text
查看 .d6a 文件中有哪些表
查看 ActionGroup 表结构
查看前几行动作帧
```

---

### 19.2 批量读取多个动作组前几帧

```bash
python3 - <<'PY'
import sqlite3
from pathlib import Path

base = Path("/home/ubuntu/software/arm_pc/config/ActionGroups")

names = [
    "init",
    "pick_init",
    "pick",
    "pick_debug",
    "hand_control_pick",
    "hand_control_place",
    "place_basket",
    "navigation_pick",
    "navigation_place",
]

for name in names:
    path = base / f"{name}.d6a"
    if not path.exists():
        continue

    conn = sqlite3.connect(str(path))
    cur = conn.cursor()
    cur.execute("SELECT * FROM ActionGroup")
    rows = cur.fetchall()
    conn.close()

    print(f"\n=== {name}.d6a ===")
    print(f"frames: {len(rows)}")
    for row in rows[:5]:
        print(row)
PY
```

用途：

```text
快速比较多个动作组的帧数和前几帧内容
观察哪些舵机变化明显
观察夹爪 Servo6 / id10 的变化
判断某个动作是否可能存在大幅运动
```
