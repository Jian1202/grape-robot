## 3. 机械臂相关核心话题

当前已确认的机械臂相关话题如下：

```text
/servo_controller
/joint_controller
/joint_states
/controller_manager/joint_states
/controller_manager/servo_states
/ros_robot_controller/bus_servo/set_position
/ros_robot_controller/bus_servo/set_state
```

初步分类：

```text
/servo_controller：上层机械臂舵机控制入口
/controller_manager/servo_states：底层舵机状态反馈
/joint_states：机械臂关节状态反馈
/joint_controller：作用暂不完全确定
/ros_robot_controller/bus_servo/set_position：底层总线舵机位置控制入口
/ros_robot_controller/bus_servo/set_state：底层总线舵机状态控制入口
/controller_manager/joint_states：controller_manager 内部关节状态
```

当前建议：

```text
优先使用 /servo_controller
优先读取 /controller_manager/servo_states
暂时不要直接操作底层 bus_servo 话题
```

## 4. 当前确认的机械臂控制链路

目前已经实验确认的控制链路：

```text
/servo_controller
↓
controller_manager
↓
servo_manager
↓
/ros_robot_controller/bus_servo/set_position
↓
ros_robot_controller
↓
底层总线舵机
↓
机械臂 / 夹爪动作
```

说明：

```text
/servo_controller 是上层接口
/ros_robot_controller/bus_servo/set_position 是底层接口
controller_manager 和 servo_manager 负责中间转换
ros_robot_controller 负责真正驱动底层硬件
```

建议后续使用方式：

```text
写控制程序时优先发布 /servo_controller
不要一开始绕过 controller_manager 直接发底层舵机命令
```

原因：

```text
/servo_controller 更接近官方代码使用方式
中间层可能包含单位转换、状态管理或安全逻辑
直接操作底层 bus_servo 接口风险更高
```

## 6. `/controller_manager/servo_states`

话题：

```text
/controller_manager/servo_states
```

作用：

```text
读取底层舵机当前状态
可以看到每个舵机 ID 当前 position
```

查看命令：

```bash
ros2 topic echo --once /controller_manager/servo_states
```

已读取到的初始状态：

```text
id1  position 500
id2  position 765
id3  position 15
id4  position 220
id5  position 500
id10 position 500
```

后续夹爪测试中观察到：

```text
id10 可以变为 540
id10 可以变为 300
id10 可以恢复为 500
```

用途：

```text
每次发布机械臂命令后，可用该话题确认底层状态是否变化
```

注意：

```text
状态变化不一定代表肉眼一定能看到明显动作
例如 id10 从 500 到 540 时，状态变了，但肉眼动作不明显
```

## 14. 安全 init 姿态命令

```bash
    {id: 2, position: 765.0},
```



## 22. 当前建议的安全控制原则

### 22.1 优先只读，再控制

每次实验前先执行：

```bash
ros2 topic echo --once /controller_manager/servo_states
```

确认当前舵机状态。

---

### 22.2 优先控制夹爪 id10

相比机械臂主体关节，夹爪小范围测试风险更低。

当前已验证安全值：

```text
id10 = 500
id10 = 540
id10 = 300
```

但仍需注意：

```text
手远离夹爪
不要夹住线缆
不要夹住相机
不要夹住易碎物
```

---

### 22.3 暂时不要大幅控制 id1~id5

原因：

```text
id1~id5 是机械臂主体关节
大幅运动可能撞到桌面、相机、机器人本体或人手
每个关节安全范围还不知道
```

---

### 22.4 暂时不要直接运行未知动作组

不要直接运行：

```text
pick.d6a
navigation_pick.d6a
navigation_place.d6a
```

原因：

```text
不知道这些动作会让机械臂移动到哪里
可能发生大幅动作
```

---

### 22.5 暂时不要直接操作底层 bus_servo

不要直接向下面的话题发布控制命令：

```text
/ros_robot_controller/bus_servo/set_position
/ros_robot_controller/bus_servo/set_state
```

除非已经明确知道消息含义和安全范围。
