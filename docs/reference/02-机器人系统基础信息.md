## 3. 远程连接方式

### 3.1 当前推荐连接方式

目前最可靠的连接方式是：

```text
手机热点 STA 局域网模式
```

网络结构：

```text
手机热点
├── ROSLander 机器人
└── Windows 电脑
```

也就是说，机器人和电脑连接到同一个手机热点，然后电脑通过 NoMachine 远程连接机器人。

---

### 3.2 当前有效机器人 IP

当前已使用过的机器人 IP：

```text
10.248.67.184
```

说明：

```text
该 IP 可能会随着重启、换热点、换网络而变化
不要把它当成永久固定 IP
```

如果连接失败，需要在机器人终端重新执行：

```bash
hostname -I
```

然后选择当前局域网中的真实 IP。

---

### 3.3 不要使用的 IP

不要使用：

```text
127.0.0.1
172.17.0.1
```

含义：

```text
127.0.0.1 是本机回环地址，只代表当前这台电脑自己
172.17.0.1 通常是 Docker 虚拟网卡地址，不用于 NoMachine 远程连接
```

NoMachine 应使用机器人在当前局域网下的真实 IP，例如：

```text
10.248.67.184
```

## 4. NoMachine 连接信息

当前 NoMachine 连接参数：

```text
Name：ROSLander
Host：10.248.67.184
Port：4000
Protocol：NX
Username：ubuntu
Password：ubuntu
```

连接步骤简要记录：

```text
1. 确认机器人和 Windows 电脑在同一个局域网
2. 在机器人终端执行 hostname -I 获取真实 IP
3. 在 NoMachine 中使用该 IP 连接
```

连接不上时，优先检查：

```text
机器人是否开机
机器人和电脑是否在同一网络
Host 是否填成了当前真实 IP
是否误用了 127.0.0.1 或 172.17.0.1
Windows 是否能 ping 通机器人 IP
NoMachine 端口是否仍为 4000
```

Windows PowerShell 中可以测试：

```powershell
ping 10.248.67.184
```

如果显示 0% 丢包，说明网络层面已连通。

## 7. ROS2 环境刷新方式

```bash

```

不要优先使用：


```text

```

每次新开终端后，如果 ROS2 命令或包找不到，可以先执行：

```bash
source ~/ros2_ws/install/setup.zsh
```

```text
/servo_manager
/joint_state_publisher
/robot_state_publisher
/init_pose
/object_tracking
/joystick_control
/kinematics
```

结论：

```text
机器人开机后不是空白 ROS2 系统
厂家预设服务已经自动启动相机、雷达、底盘、机械臂等相关节点
```

---

### 8.2 不要重复启动 bringup

重启后不要一上来就执行：

```bash
ros2 launch bringup bringup.launch.py
```

应先检查：

```bash
ros2 node list
```

原因：

```text
核心节点可能已经自动启动
重复启动可能造成节点冲突
可能影响相机、底盘或机械臂控制
```

是否所有情况下都不能手动启动 bringup：

```text
不知道
```

当前安全策略：

```text
先检查节点
确认没有相关节点运行时，再考虑是否需要启动 bringup
```

## 9. 默认 App 服务

机器人存在默认 App 服务，可能会与实验程序冲突。

服务名称：

```text

```

停止默认 App 服务：

```bash
sudo systemctl stop start_app_node.service
```

恢复默认 App 服务：

```bash
sudo systemctl start start_app_node.service
```

查看服务状态：

```bash
systemctl status start_app_node.service
```


## 15. 每次连接机器人后的推荐检查流程

进入 NoMachine 后，先不要启动 bringup。

建议执行：

```bash
hostname -I
echo $ROS_VERSION
echo $ROS_DISTRO
source ~/ros2_ws/install/setup.zsh
ros2 node list
```

然后检查关键话题：

```bash
ros2 topic list | grep -E "scan|camera|cmd_vel|odom|joint|battery|servo|controller"
```

如果要运行抓取程序，再确认是否使用安全脚本：

```bash
bash ~/run_ros2_track_and_grab.sh green
```

不要跳过脚本里的安全确认。
