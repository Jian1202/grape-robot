## 3. 远程连接方式

### 3.1 当前推荐连接方式

目前最可靠的连接方式是：

```text
手机热点 STA 局域网模式
```

网络结构：

```text
手机热点
├── ROSLander 机器人
└── Windows 电脑
```

也就是说，机器人和电脑连接到同一个手机热点，然后电脑通过 NoMachine 远程连接机器人。

---

### 3.2 当前有效机器人 IP

当前已使用过的机器人 IP：

```text
10.248.67.184
10.248.67.8
```

说明：

```text
该 IP 可能会随着重启、换热点、换网络而变化
不要把它当成永久固定 IP
```

如果连接失败，需要在机器人终端重新执行：

```bash
hostname -I
```

然后选择当前局域网中的真实 IP。

---

### 3.3 不要使用的 IP

不要使用：

```text
127.0.0.1
127.0.1.1
172.17.0.1
```

含义：

```text
127.0.0.1 / 127.0.1.1 是本机回环地址，只代表机器人自己
172.17.0.1 通常是 Docker 虚拟网卡地址，不用于 NoMachine 或 SSH 远程连接
```

NoMachine 和 SSH 都应使用机器人在当前局域网下的真实 IP，例如：

```text
10.248.67.8
```

---

### 3.4 使用网卡状态查询真实 IP

如果 `hostname -I` 只显示：

```text
127.0.1.1
```

或者看起来没有可用的局域网 IP，可以在机器人本机终端执行：

```bash
ip -br addr
```

再执行：

```bash
nmcli device status
```

作用：

```text
ip -br addr：查看每个网卡当前拿到的 IP 地址
nmcli device status：查看每个网络设备是否 connected / disconnected / unavailable
```

常见 Wi-Fi 网卡名可能是：

```text
wlp1p1s0
wlan0
```

如果看到类似：

```text
wlp1p1s0    UP    10.248.67.8/24
```

说明机器人当前真实局域网 IP 是：

```text
10.248.67.8
```

此时电脑应该使用：

```bash
ssh ubuntu@10.248.67.8
```

或在 NoMachine 中把 Host 改成：

```text
10.248.67.8
```

如果看到：

```text
docker0    172.17.0.1/16
```

不要使用它远程连接机器人。

如果 `nmcli device status` 里 Wi-Fi 设备显示：

```text
connected
```

说明 Wi-Fi 已连接。

如果显示：

```text
disconnected
unavailable
```

说明 Wi-Fi 没有正常连上，需要重新连接手机热点或检查网络模式。

## 4. NoMachine 连接信息

当前 NoMachine 连接参数：

```text
Name：ROSLander
Host：当前真实 IP，例如 10.248.67.8
Port：4000
Protocol：NX
Username：ubuntu
Password：ubuntu
```

连接步骤简要记录：

```text
1. 确认机器人和 Windows 电脑在同一个局域网
2. 在机器人终端执行 hostname -I 获取真实 IP
3. 如果 hostname -I 只显示 127 开头地址，执行 ip -br addr 和 nmcli device status
4. 在 NoMachine 中使用真实局域网 IP 连接
```

连接不上时，优先检查：

```text
机器人是否开机
机器人和电脑是否在同一网络
Host 是否填成了当前真实 IP
是否误用了 127.0.0.1、127.0.1.1 或 172.17.0.1
Windows 是否能 ping 通机器人 IP
NoMachine 端口是否仍为 4000
```

Windows PowerShell 中可以测试：

```powershell
ping 10.248.67.8
```

如果显示 0% 丢包，说明网络层面已连通。

## 5. SSH 连接信息

SSH 用于从电脑终端直接进入机器人 Ubuntu 命令行。

它适合：

```text
查看源码
执行 ROS2 只读检查命令
运行脚本
复制文件
让 agent 通过命令行分析项目
```

它不适合：

```text
查看机器人图形界面
直接看摄像头窗口
操作需要桌面的软件
```

图形界面优先使用 NoMachine，命令行操作优先使用 SSH。

---

### 5.1 普通密码 SSH 连接

在 Windows PowerShell、Mac 终端或 Linux 终端中执行：

```bash
ssh ubuntu@10.248.67.8
```

然后输入机器人密码：

```text
ubuntu
```

含义：

```text
ubuntu：机器人用户名
10.248.67.8：机器人当前局域网 IP
```

如果机器人 IP 变化，需要把命令中的 IP 替换为新的真实 IP。

---

### 5.2 使用 SSH key 连接

如果已经生成过 SSH key，并且机器人已经配置了对应公钥，可以使用：

```bash
ssh -i ~/.ssh/roslander_ed25519 ubuntu@10.248.67.8
```

如果创建 key 时设置过口令，连接时会提示输入 key 的 passphrase。

说明：

```text
-i ~/.ssh/roslander_ed25519：指定本机使用哪一个私钥
ubuntu@10.248.67.8：以 ubuntu 用户连接机器人当前 IP
```

---

### 5.3 SSH 连接失败时的检查

先确认网络是否连通。

在电脑终端执行：

```bash
ping 10.248.67.8
```

如果 ping 不通，优先检查：

```text
机器人是否开机
电脑和机器人是否连接同一个热点 / 局域网
机器人 IP 是否变化
是否误用了 127.0.0.1、127.0.1.1 或 172.17.0.1
```

如果 IP 可能变化，在机器人本机或 NoMachine 终端执行：

```bash
hostname -I
```

如果 `hostname -I` 只显示 127 开头地址，再执行：

```bash
ip -br addr
nmcli device status
```

找到 Wi-Fi 网卡对应的真实 IP 后，用新的 IP 重新连接：

```bash
ssh ubuntu@新的IP
```

---

### 5.4 SSH 连接成功后的推荐第一组命令

进入机器人后，先执行：

```bash
hostname -I
ip -br addr
nmcli device status
echo $ROS_VERSION
echo $ROS_DISTRO
echo $SHELL
source ~/ros2_ws/install/setup.zsh
ros2 node list
```

这组命令用于确认：

```text
当前机器人 IP
当前网卡连接状态
当前 ROS 版本
当前 shell 类型
ROS2 环境是否可用
当前自动启动了哪些 ROS2 节点
```

注意：

```text
SSH 登录后不要一上来启动 bringup
不要直接运行会控制底盘或机械臂的程序
先检查节点和话题
```

### 9.2 不要重复启动 bringup

重启后不要一上来就执行：

```bash
ros2 launch bringup bringup.launch.py
```

```bash
ros2 node list
```

原因：

```text
核心节点可能已经自动启动
重复启动可能造成节点冲突
可能影响相机、底盘或机械臂控制
```

```text
不知道
```

当前安全策略：

```text
先检查节点
确认没有相关节点运行时，再考虑是否需要启动 bringup
```

```bash
find ~/ros2_ws/src -name "track_and_grab.py"
sed -n '1,260p' 文件路径
grep -R "关键词" -n ~/ros2_ws/src/example | head -300
```

不要让 agent 执行：

```text
控制底盘命令

发布 ROS2 控制话题
启动 bringup
运行抓取程序

sudo shutdown
apt install

```

## 17. 当前已知问题与注意事项

### 17.1 IP 可能变化

`10.248.67.184` 和 `10.248.67.8` 都只是曾经使用过的有效 IP，不是永久固定值。

解决方式：

```bash
hostname -I
ip -br addr
nmcli device status
```

---

### 17.2 hostname -I 可能只显示本地地址

如果只显示：

```text
127.0.1.1
```

不要使用它 SSH。

继续执行：

```bash
ip -br addr
nmcli device status
```

从 Wi-Fi 网卡上找真实局域网 IP。

---

### 17.3 Docker 虚拟网卡不要用于远程连接

`172.17.0.1` 不用于 NoMachine 或 SSH 连接。

---

### 17.4 不要重复启动核心节点

机器人可能已经自动启动核心节点。

启动前先看：

```bash
ros2 node list
```

---

### 17.5 setup.bash 报错不一定是环境坏了

当前 shell 是 zsh，应优先：

```bash
source ~/ros2_ws/install/setup.zsh
```

---

### 17.6 ROS1 命令不能直接照搬

当前为 ROS2 Humble。

不要直接使用：

```bash
roscore
roslaunch
rosrun
rostopic
rosservice
```

---

### 17.7 默认 App 服务可能与实验冲突

运行抓取程序前要注意：

```text
start_app_node.service
```

优先使用官方/已整理的安全脚本启动实验。

---

### 17.8 SSH 适合命令行，不适合图形界面

SSH 适合查看代码、执行命令、让 agent 分析工程。

如果需要看桌面、摄像头窗口或图形化工具，应使用：

```text
NoMachine
```

## 19. 一句话总结

ROSLander 当前已能通过 NoMachine 远程桌面和 SSH 命令行连接；机器人真实 IP 应优先通过 `hostname -I` 查看，若只显示 127 开头地址，则使用 `ip -br addr` 和 `nmcli device status` 结合判断 Wi-Fi 网卡真实 IP；后续实验前应先检查节点与话题，避免重复启动 bringup、误用虚拟网卡 IP，或与默认 App 服务冲突。
