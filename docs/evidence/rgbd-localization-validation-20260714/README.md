## 1. 验证目标

本轮只验证以下链路：

```text
YOLO检测框
-> 深度像素反投影到深度相机坐标
-> depth_to_color外参变换
-> 投影到RGB检测框
-> 框内深度覆盖率与稳健深度
-> 仅显示结果或明确失败
```

本轮不验证：

- 机械臂跟踪、IK动作和夹爪深入距离；
- 夹爪闭合、保持、归位；
- 向下脱离；
- 机械臂或底盘主动搜索；
- 手眼矩阵和机械臂世界坐标精度。

这些项目必须等待本文件中的Gate 2、Gate 3通过后另行验证。

本文件采用“关键路径先行”原则：

1. 默认只验证会决定能否继续的最少项目；
2. 关键项全部通过后就停止，不为了补表格执行全部边界场景；
3. 某个关键项失败后，才增加与该症状相关的细化验证；
4. 细化验证的目标是定位原因，不是扩大测试数量。

## 2. 安全边界

被测定位程序必须保持：

```text
enable_arm=false
不发布 /servo_controller
不创建或调用运动学服务客户端
不发布 /cmd_vel 或 /controller/cmd_vel
不调用夹爪、机械臂、底盘动作服务
不修改舵机pulse
```

当前代码还有额外硬保护：传入`enable_arm=true`时节点拒绝启动，并且检测-only构造器不创建执行器publisher或运动学client。

出现以下任一情况立即停止验证：

- 节点出现在`/servo_controller`发布者列表中；
- 机械臂、夹爪或底盘发生任何非预期动作；
- 实际运行代码与记录的Git提交不一致；
- RGB-D元数据缺失但程序仍输出成功定位；
- 目标消失后仍显示上一帧成功位置；
- 现场人员不能立即终止程序。

### 5.3 暂定验收标准

这些标准仅用于Gate 2/3感知筛选，不代表机械臂抓取精度：

| 指标 | 暂定通过条件 |
| --- | --- |
| 投影位置 | 连续帧投影点位于葡萄主体区域，不落在明显背景或叶片上 |
| 距离误差 | 与沿相机光轴测得的葡萄表面距离相比，绝对误差不大于0.03m |
| 静止抖动 | 同一静止场景10帧深度最大值减最小值不大于0.02m |
| 有效覆盖 | 正常完整目标不低于`min_valid_ratio=0.15` |
| 失败关闭 | 元数据缺失、空框、有效点不足时100%返回失败，不输出旧目标 |
| 可重复性 | 同一固定样本重复运行3次结果一致 |

如果实测误差系统性超过3cm，不要修改手眼偏移或夹爪深入量补偿；先检查相机内参、外参、测量方法和深度层选择。
